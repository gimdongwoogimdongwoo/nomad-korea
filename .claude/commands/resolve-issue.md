당신은 GitHub 이슈 작성 전문가입니다.
아래 요청을 분석해 잘 구조화된 GitHub 이슈를 생성하세요.
모든 이슈 본문은 한국어로 작성하세요.

**요청:** $ARGUMENT

---

## Step 1 — 컨텍스트 파악

다음을 순서대로 확인하세요.

1. `SPEC.md`가 있으면 읽어 프로젝트 맥락과 Phase 진행 상황을 파악하세요.
2. `git log --oneline -5` 로 최근 커밋을 확인하세요.
3. 요청을 아래 유형 중 하나로 분류하세요.
   - **bug**: 기존 동작이 의도와 다름
   - **feature**: 새로운 기능 추가
   - **enhancement**: 기존 기능 개선
   - **chore**: 리팩터링, 설정, 의존성 업데이트
   - **docs**: 문서 작성/수정

---

## Step 2 — 이슈 초안 작성

아래 구조로 이슈를 작성하세요.

### 제목
- 50자 이내, 한국어
- 형식: `[유형] 무엇을 한다` (예: `[feature] 도시 상세 페이지 404 커스텀 처리`)

### 본문

```
## 개요
<!-- 이 이슈가 해결하는 문제 또는 추가하는 가치를 1~2문장으로 -->

## 작업 내용
- [ ] 항목 1
- [ ] 항목 2
- [ ] ...

## 완료 기준
- [ ] (사용자가 확인할 수 있는 동작 기준)
- [ ] lint & typecheck 통과

## 관련 파일 / 기술 메모
<!-- 관련 파일 경로, 주의사항, 참고 링크 등 (없으면 생략) -->
```

### 레이블
유형에 따라 하나 선택: `bug`, `enhancement`, `feature`, `chore`, `docs`

---

## Step 3 — 이슈 생성

초안이 완성되면 `gh issue create` 명령어로 실제 이슈를 생성하세요.

```bash
gh issue create \
  --title "<제목>" \
  --body "$(cat <<'EOF'
<본문>
EOF
)" \
  --label "<레이블>"
```

생성 완료 후 반환된 이슈 URL을 사용자에게 출력하세요.

GitHub remote가 설정되지 않은 경우 초안만 출력하고, 이슈를 생성하려면 `gh repo create` 또는 remote 설정이 필요하다고 안내하세요.
